# Sodiq School — Diagnostic Reports (web + A4 PDF)

Premium per-student academic diagnostic reports for **three entrance exams**
(Matematika, Ingliz tili, Tanqidiy fikrlash) plus a cross-subject **summary
dashboard**. Every indicator is **computed from per-question labels**
(`questions[]`) — nothing is hand-set. Navy/orange brand, Uzbek Latin (no
Cyrillic, ASCII apostrophes), responsive web + A4 PDF export.

## Pages

| Route | Page |
|---|---|
| `/summary` | Cross-subject dashboard (composite + 3 subject cards + insights) |
| `/` | Matematika report |
| `/english` | Ingliz tili report (CEFR A1) |
| `/critical-thinking` | Tanqidiy fikrlash report |

## Quick start

```bash
npm install
npm run dev            # http://localhost:4321
npm run build          # static output → dist/
```

## Deploy to Vercel

The site is a **static Astro build** — Vercel auto-detects the framework
(build `astro build`, output `dist/`). `vercel.json` only overrides the install
step to skip Playwright's Chromium download (Playwright is dev-only, for PDF
export; the deployed site never needs it).

1. Push this repo to GitHub.
2. In Vercel: **New Project → Import** the repo → Deploy (no settings needed).

## How it works

```
src/data/student.json            Matematika  — meta + questions[] (labels)
src/data/english.json            Ingliz tili — generated by scripts/gen-english.mjs
src/data/critical-thinking.json  Tanqidiy fikrlash — generated by scripts/gen-ct.mjs
src/lib/compute.js               the engine — derives EVERY indicator from questions[]
src/lib/programs*.js             3/6/12-month roadmap templates wired to computed numbers
src/lib/svg.js                   hand-built SVG instruments (hero, road, dumbbell, radar, …)
src/pages/*.astro                summary + one page per subject
src/components/StageDetail.astro on-demand per-stage roadmap dossier
src/styles/tokens.css            design tokens (fonts, colour/type/space/radius/shadow)
src/styles/global.css            components, built only from tokens; print + responsive
public/                          self-hosted fonts + logo
scripts/                         data generators · PDF export (Playwright) · verify-compute
```

Every report is driven by one labelled `questions[]` array. The same
`compute.js` engine works across subjects (it groups by `strand`, `topic`,
`skill`, `bloom`, `gradeLevel`, `difficulty`). Cohort/percentile values are
placeholders until real cohort data exists.

## Regenerating data

The English and Critical-Thinking datasets are produced from their real exam
taxonomies + a simulated student profile:

```bash
node scripts/gen-english.mjs     # → src/data/english.json
node scripts/gen-ct.mjs          # → src/data/critical-thinking.json
```

Swap in a real student by editing the `WRONG`/`PARTIAL` sets (or the per-question
arrays) in those scripts, or replace the JSON directly.

## PDF export (local only)

```bash
npx playwright install chromium  # first run only
npm run export:pdf               # build → preview → render A4 PDF
```

## Design system

Modern-premium "instrument" look, all driven by `src/styles/tokens.css`: an 8px
spacing grid, one type scale, three radii, a soft elevation scale, and a
navy + orange brand with green/amber/red data semantics. Charts are hand-built
inline SVG (cover hero, the bike **roadmap**, the **growth dumbbell**, the skill
**radar**, the **tier-units** grid) plus one ECharts line for the forecast.
Animations are disabled under `prefers-reduced-motion` and in print, so the PDF
shows the final static state. Print has a real A4 pass (`@page`,
`print-color-adjust`, page-break control).

> Fonts are self-hosted in `public/fonts/`; the bundled Pragmatica is a converted
> webfont — confirm a proper licence before production.
